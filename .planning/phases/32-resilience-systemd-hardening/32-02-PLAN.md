  <action>
  Add a `_check_consistency(db_path)` function to each service's main.py, called in `main()` between `init_db()` and `BaseService(...).run()`. Each check is service-specific:

  **Extractor (services/extractor/main.py):**
  Detect papers with stage="analyzed" that have 0 formulas (might indicate a failed extraction that didn't complete). Log a warning but do NOT auto-fix — extraction is expensive and should be triggered explicitly.
  ```python
  def _check_consistency(db_path: str) -> None:
      conn = get_connection(db_path)
      try:
          stuck = conn.execute("""
              SELECT p.id, p.arxiv_id FROM papers p
              WHERE p.stage = 'extracted'
              AND NOT EXISTS (SELECT 1 FROM formulas f WHERE f.paper_id = p.id)
          """).fetchall()
          if stuck:
              logger.warning(
                  "Consistency: %d papers at stage 'extracted' with 0 formulas: %s",
                  len(stuck), [row[1] for row in stuck]
              )
      finally:
          conn.close()
  ```

  **Validator (services/validator/main.py):**
  Detect formulas with stage="extracted" that have partial validations (some engines completed, others not). This can happen if validator crashes mid-batch. Log warning with counts.
  ```python
  def _check_consistency(db_path: str) -> None:
      conn = get_connection(db_path)
      try:
          partial = conn.execute("""
              SELECT f.id, f.latex_hash, COUNT(v.id) as validation_count
              FROM formulas f
              LEFT JOIN validations v ON v.formula_id = f.id
              WHERE f.stage = 'extracted'
              AND v.id IS NOT NULL
              GROUP BY f.id
          """).fetchall()
          if partial:
              logger.warning(
                  "Consistency: %d formulas have partial validations (stage still 'extracted'): %s",
                  len(partial), [(row[0], row[2]) for row in partial[:10]]
              )
      finally:
          conn.close()
  ```

  **Codegen (services/codegen/main.py):**
  Detect formulas with stage="validated" that have partial generated_code (e.g., Python generated but not Rust). Log warning.
  ```python
  def _check_consistency(db_path: str) -> None:
      conn = get_connection(db_path)
      try:
          partial = conn.execute("""
              SELECT f.id, f.latex_hash, COUNT(g.id) as code_count
              FROM formulas f
              LEFT JOIN generated_code g ON g.formula_id = f.id
              WHERE f.stage = 'validated'
              AND g.id IS NOT NULL
              GROUP BY f.id
          """).fetchall()
          if partial:
              logger.warning(
                  "Consistency: %d validated formulas have partial codegen: %s",
                  len(partial), [(row[0], row[2]) for row in partial[:10]]
              )
      finally:
          conn.close()
  ```

  Each function:
  - Is called once at startup, before service.run()
  - Only logs warnings, does NOT modify data (safe, non-destructive)
  - Catches and logs exceptions (DB errors should not prevent startup)
  - Wraps the whole function body in try/except with logger.error
