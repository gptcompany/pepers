  <name>Task 1: Schema migration v2→v3 with UNIQUE constraint</name>
  <files>shared/db.py</files>
  <action>
  Add a real migration framework to shared/db.py. Current state: init_db() uses CREATE IF NOT EXISTS + INSERT OR IGNORE for version tracking — no actual migration logic.

  Implementation:
  1. Add a `MIGRATIONS` dict mapping version numbers to SQL strings:
     ```python
     MIGRATIONS: dict[int, str] = {
         3: """
         -- Add UNIQUE constraint on (paper_id, latex_hash)
         -- SQLite cannot ALTER TABLE ADD CONSTRAINT, so we:
         -- 1. Create new table with constraint
         -- 2. Copy data (dedup by keeping lowest id per paper_id+latex_hash)
         -- 3. Drop old table
         -- 4. Rename new table
         CREATE TABLE formulas_v3 (
             id INTEGER PRIMARY KEY AUTOINCREMENT,
             paper_id INTEGER NOT NULL REFERENCES papers(id),
             latex TEXT NOT NULL,
             latex_hash TEXT NOT NULL,
             description TEXT,
             formula_type TEXT,
             context TEXT,
             stage TEXT NOT NULL DEFAULT 'extracted',
             error TEXT,
             created_at TEXT NOT NULL DEFAULT (datetime('now')),
             UNIQUE(paper_id, latex_hash)
         );
         INSERT OR IGNORE INTO formulas_v3 (id, paper_id, latex, latex_hash, description, formula_type, context, stage, error, created_at)
             SELECT id, paper_id, latex, latex_hash, description, formula_type, context, stage, error, created_at
             FROM formulas
             GROUP BY paper_id, latex_hash
             HAVING id = MIN(id);
         DROP TABLE formulas;
         ALTER TABLE formulas_v3 RENAME TO formulas;
         """
     }
     ```
  2. Add `_run_migrations(conn)` function that:
     - Reads current max version from schema_version table
     - Applies each migration > current version in order
     - Inserts new version row after each successful migration
     - Wraps each migration in a transaction
  3. Call `_run_migrations(conn)` at the end of `init_db()` after the existing CREATE TABLE IF NOT EXISTS block
  4. Re-create the indexes after migration (they're dropped with the old table):
     - `idx_formulas_paper_id`
     - `idx_formulas_latex_hash`

  IMPORTANT: Preserve backward compatibility — existing databases with schema v2 must migrate cleanly. New databases get the UNIQUE constraint from the initial CREATE TABLE (update the SCHEMA constant too).

  Also update the SCHEMA constant's formulas CREATE TABLE to include the UNIQUE constraint for fresh databases.

  <action>
  **Docker (docker-compose.yml):**
  Add `restart: unless-stopped` to all 6 service definitions. Currently missing — Docker won't restart crashed containers.

  **systemd (deploy/ directory):**
  Create systemd user unit files for standalone (non-Docker) deployment. One .service per microservice + one .target to group them.

  Template for each service (example: rp-validator.service):
  ```ini
  [Unit]
  Description=Research Pipeline - Validator Service
  After=network.target
  PartOf=rp-pipeline.target

  [Service]
  Type=simple
  WorkingDirectory=/media/sam/1TB/research-pipeline
  Environment=RP_DB_PATH=data/research.db
  Environment=RP_VALIDATOR_PORT=8773
  ExecStart=/media/sam/1TB/research-pipeline/.venv/bin/python -m services.validator.main
  Restart=always
  RestartSec=5
  WatchdogSec=60

  # Hardening
  PrivateTmp=true
  NoNewPrivileges=true
  ProtectSystem=strict
  ReadWritePaths=/media/sam/1TB/research-pipeline/data

  [Install]
  WantedBy=rp-pipeline.target
  ```

  Port mapping for services:
  - discovery: 8770
  - analyzer: 8771
  - extractor: 8772
  - validator: 8773
  - codegen: 8774
  - orchestrator: 8775

  The rp-pipeline.target:
  ```ini
  [Unit]
  Description=Research Pipeline - All Services
  Wants=rp-discovery.service rp-analyzer.service rp-extractor.service rp-validator.service rp-codegen.service rp-orchestrator.service

  [Install]
  WantedBy=default.target
  ```

  Do NOT add sd_notify integration to the Python code — that would require the `systemd` Python package and add complexity. Type=simple with WatchdogSec + Restart=always is sufficient. If the process dies, systemd restarts it. WatchdogSec detects hangs via process liveness (not application-level).
  </action>
