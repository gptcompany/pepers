**File**: `shared/config.py`

Add module-level constant with error handling:
```python
def _parse_float_env(name: str, default: str) -> float:
    raw = os.environ.get(name, default)
    try:
        return float(raw)
    except ValueError:
        logger.warning("%s invalid value '%s', using default %s", name, raw, default)
        return float(default)
```

2. `call_gemini_cli()` ‚Äî **Limitation**: Gemini CLI (`gemini -p`) does NOT support `--temperature` flag (verified). Temperature cannot be controlled via CLI. Document this known limitation. No code change needed for this provider.
3. `call_gemini_sdk()` ‚Äî replace hardcoded `temperature=0.3` with `temperature` parameter defaulting to `LLM_TEMPERATURE`. Add `seed=42` param for determinism (Gemini SDK GenerateContentConfig supports seed).
4. `call_openrouter()` ‚Äî replace hardcoded `"temperature": 0.3` with `temperature` parameter defaulting to `LLM_TEMPERATURE`. Add `"seed": 42` for reproducibility.
5. `call_ollama()` ‚Äî replace hardcoded `{"temperature": 0.3}` default with `LLM_TEMPERATURE`. Add `"seed": 42` to default options.
6. `fallback_chain()` ‚Äî add `temperature: float | None = None` parameter, thread it to each provider call via kwargs.

**Signature changes**:
```python

def call_openrouter(prompt, system, model=..., timeout=..., temperature=LLM_TEMPERATURE):

```

For `call_ollama`, if `options` is None, build default options using the `temperature` parameter. If `options` is provided explicitly, respect it (caller override).

For `fallback_chain`, pass temperature to each provider via keyword arg.
