### Task 3: Implement `services/orchestrator/pipeline.py`

Core pipeline dispatch logic.

```python
# ~200 LOC
# - STAGE_ORDER: list of (name, port) tuples
# - PipelineRunner class:
#   - __init__(db_path, timeout, retry_max, retry_backoff)
#   - run(query, paper_id, stages, max_papers, max_formulas, force) -> dict
#   - _dispatch_stage(name, port, params) -> dict  (with retry)
#   - _call_service_with_retry(url, params) -> dict
#   - _generate_run_id() -> str
#   - _get_paper_stage(paper_id) -> str
#   - get_pipeline_status() -> dict  (aggregate DB query)
#   - get_services_health() -> dict  (call each /health)
```

**Key behaviors:**
- Sequential dispatch through STAGE_ORDER
- Retry: 3 retries, backoff 4^attempt (1s, 4s, 16s)
- Per-stage error: log, record in results, continue to next stage
- Run status: "completed" if all stages OK, "partial" if any failed
- Parameter forwarding per the DESIGN.md mapping table
- Uses `requests.post()` with configurable timeout
