# Plan 36-01: GET /generated-code + Async /run + Pipeline Runs Table

## Goal

Add GET /generated-code endpoint, make POST /run async (HTTP 202 + polling), persist run records in pipeline_runs table.

## Tasks

### Task 1: Schema Migration v4 (pipeline_runs)

**File**: `shared/db.py` (~15 LOC)

Add migration v4 to MIGRATIONS dict:
```sql
CREATE TABLE IF NOT EXISTS pipeline_runs (
    run_id TEXT PRIMARY KEY,
    status TEXT NOT NULL DEFAULT 'running',
    params TEXT,
    results TEXT,
    errors TEXT,
    stages_completed INTEGER DEFAULT 0,
    stages_requested INTEGER DEFAULT 0,
    started_at TEXT NOT NULL DEFAULT (datetime('now')),
    completed_at TEXT
);
```

### Task 2: Run Persistence in PipelineRunner

**File**: `services/orchestrator/pipeline.py` (~50 LOC)

- `_create_run_record(run_id, params, stages_requested)` — INSERT into pipeline_runs
- `_update_run_record(run_id, result)` — UPDATE status, results, errors, completed_at
- `get_run_status(run_id)` → dict | None — SELECT single run
- `list_runs(limit=20)` → list[dict] — SELECT recent runs ORDER BY started_at DESC
- Modify `run()` to accept optional `run_id` param and persist

### Task 3: GET /generated-code

**File**: `services/orchestrator/main.py` (~40 LOC)

New route:
- Parse query params: paper_id (required), language, formula_id, limit, offset
- JOIN generated_code + formulas for latex/description
- Return list of code objects

### Task 4: Async POST /run + GET /runs

**File**: `services/orchestrator/main.py` (~40 LOC)

- `handle_run()`: generate run_id, create run record, spawn thread, send HTTP 202
- New `@route("GET", "/runs")`: dispatch to list or single based on `?id=` param
- `_run_pipeline_async(runner, run_id, **params)`: thread target, calls runner.run()
