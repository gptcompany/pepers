# Plan 27-01: GitHub Discovery Testing

## Overview

Unit, integration, and E2E tests for the GitHub Discovery feature (Phase 26). Focus on E2E with real GitHub API and Gemini CLI analysis. All API keys and tools verified available.

## Depends On

- Phase 26 implementation: `services/orchestrator/github_search.py` (621 LOC)
- Phase 26 endpoints: `services/orchestrator/main.py` (POST /search-github, GET /github-repos)
- Phase 26 models: `shared/models.py` (GitHubRepo, GitHubAnalysis, SearchGitHubRequest, SearchGitHubResponse)
- Existing test infrastructure: conftest.py fixtures (`initialized_db`, `tmp_db_path`, `clean_env`, `sample_paper_row`)

## Tasks

### Task 1: Unit Tests (`tests/unit/test_github_search.py`)

**What**: Test all pure functions in `github_search.py` without external deps.

**Functions to test**:
- `_extract_keywords(title)` — stop word removal, max 5, min length 3
- `_get_github_headers()` — PAT from RP_GITHUB_PAT, fallback GITHUB_PAT, no PAT
- `_check_rate_limit(resp_headers)` — sleep when remaining < 2, no-op otherwise
- `build_dynamic_prompt(paper_context, repo_info)` — prompt construction with/without formulas
- `_parse_json_response(text)` — raw JSON, markdown-fenced JSON, invalid JSON
- `_read_repo_files(repo_path)` — file reading with extensions, skip dirs, max_chars limit
- `_generate_queries(paper_context)` — query generation from title keywords
- `clone_repo()` + `cleanup_clone()` — basic clone/cleanup (mocked subprocess)

**LOC**: ~250

### Task 2: Integration Tests (`tests/integration/test_github_search_db.py`)

**What**: Test DB operations and HTTP endpoints with real SQLite, mocked GitHub/Gemini.

**Test groups**:
- `_store_repo()` — INSERT OR IGNORE, UNIQUE constraint, return ID
- `_store_analysis()` — INSERT analysis, JSON fields serialization
- `_load_paper_context()` — paper + formulas loading, nonexistent paper
- `search_and_analyze()` — full flow with mocked `search_github` + `analyze_with_gemini_cli`
- POST /search-github endpoint — validation, happy path, paper not found
- GET /github-repos endpoint — query by paper_id, filter by recommendation, limit

**LOC**: ~350

### Task 3: E2E Tests (`tests/e2e/test_github_search_e2e.py`)

**What**: Real GitHub API search + real Gemini CLI analysis on actual repos. Uses GITHUB_PAT and GEMINI_API_KEY from dotenvx.

**Test cases**:
- `test_search_github_real_api` — search for "kelly criterion" repos, verify results structure
- `test_clone_and_read_real_repo` — clone a known small repo, verify file reading
- `test_analyze_with_gemini_cli_real` — analyze a real cloned repo with Gemini CLI
- `test_full_search_and_analyze_e2e` — complete flow: DB seed → search → analyze → verify DB records
- `test_search_github_endpoint_e2e` — HTTP POST /search-github with real server + real API
- `test_github_repos_endpoint_after_search` — HTTP GET /github-repos after successful search

**Environment**: Tests load GITHUB_PAT and GEMINI_API_KEY via `dotenvx run`. Skip if not available.

**LOC**: ~300

### Task 4: Conftest Fixtures (`tests/conftest.py`)

**What**: Add shared fixtures for GitHub Discovery tests.

**New fixtures**:
- `sample_github_repo_data` — sample repo dict
- `sample_github_analysis_data` — sample analysis dict
- `github_paper_db` — DB with a seeded paper ready for GitHub search

**LOC**: ~40

## Execution Order

1. Task 4 (fixtures) — foundation, no dependencies
2. Task 1 (unit tests) — uses fixtures
3. Task 2 (integration tests) — uses fixtures + real DB
4. Task 3 (E2E tests) — uses everything + real APIs

## Risk Assessment

- **Low**: Unit tests — pure functions, well-defined input/output
- **Low**: Integration tests — follows established patterns from 6 milestones
- **Medium**: E2E GitHub API — rate limits, network dependency
- **Medium**: E2E Gemini CLI — model availability, response parsing variability
- **Mitigation**: E2E tests use `pytest.mark.skipif` when APIs unavailable, generous timeouts
