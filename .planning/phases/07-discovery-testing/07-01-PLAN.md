## Test Plan

### Unit Tests (`tests/unit/test_discovery.py`)

**TestExtractArxivId** (~5 tests):
- Versioned ID (e.g. `http://arxiv.org/abs/2107.05580v2` → `2107.05580`)
- Unversioned ID (e.g. `http://arxiv.org/abs/2107.05580` → `2107.05580`)
- New-style ID (e.g. `http://arxiv.org/abs/cs/0112017v1` → `cs/0112017`)
- Multi-digit version (`v12` → stripped)

**TestSearchArxiv** (~6 tests) — mock `arxiv.Client` + `arxiv.Search`:
- Returns paper dicts with correct fields
- Filters DataCite DOIs (prefix 10.48550 → None)
- Keeps journal DOIs (non-DataCite)
- Empty results returns empty list
- Authors and categories serialized as JSON strings
- Published date as ISO format

**TestEnrichS2** (~7 tests) — mock `requests.get`:
- 200 success with full data → enrichment dict
- 200 with missing optional fields → defaults
- 200 with journal DOI in externalIds → doi included
- 200 with DataCite DOI in externalIds → doi excluded
- 404 → returns None
- 429 first attempt → retry with Retry-After → 200 success
- 429 both attempts → returns None
- Request exception → returns None

**TestEnrichCrossref** (~6 tests) — mock `requests.get`:
- 200 success → returns message dict
- 404 → returns None
- 429 → retry → 200 success
- 429 → retry → still fails → returns None
- Request exception → returns None
- Response structure: message extracted from data

**TestUpsertPaper** (~5 tests) — mock `transaction`:
- New paper → returns paper_id
- Existing paper (same arxiv_id) → returns updated paper_id
- Missing optional fields → uses defaults
- DB error → returns None, logs error

**TestUpdatePaperS2** (~4 tests) — mock `transaction`:
- Standard update → returns True
- Update with DOI → includes doi column
- Update without DOI → excludes doi column
- DB error → returns False

**TestUpdatePaperCrossref** (~3 tests) — mock `transaction`:
- Valid update → returns True
- JSON serialization of crossref_data
- DB error → returns False

**TestDiscoveryHandlerValidation** (~4 tests) — mock handler context:
- Missing query → 422 VALIDATION_ERROR
- Empty query string → 422
- Invalid max_results (0, 501, string) → 422
- Valid defaults (no max_results → DEFAULT_MAX_RESULTS)

### Integration Tests (`tests/integration/test_discovery_db.py`)

Uses `initialized_db` fixture (real SQLite on tmp_path), mock HTTP APIs:

**TestUpsertPaperDB** (~5 tests):
- Insert new paper → verify in DB
- Upsert existing → verify updated fields, same id
- Upsert preserves created_at, updates updated_at
- Multiple papers → unique IDs
- Paper with NULL optional fields

**TestUpdatePaperS2DB** (~3 tests):
- Update S2 fields → verify all fields in DB
- Update with DOI → doi column updated
- Update non-existent paper_id → still executes (no FK on id for UPDATE)

**TestUpdatePaperCrossrefDB** (~2 tests):
- Store CrossRef JSON blob → verify stored as JSON string
- Verify JSON round-trip (store → read → parse)

**TestDiscoveryHandlerIntegration** (~5 tests) — real server + real DB, mock arXiv/S2/CR:
- Full pipeline: mock arXiv returns 2 papers, mock S2/CR → papers in DB
- Duplicate run → papers_new=0, data updated
- arXiv returns empty → papers_found=0
- S2 fails for one paper → partial enrichment, error logged
- CrossRef enrichment only for papers with journal DOI

### E2E Tests (`tests/e2e/test_discovery_e2e.py`)

Real API calls, `@pytest.mark.e2e`, skipped in CI:

- Search arXiv for Kelly criterion papers → real results
- Enrich with Semantic Scholar → real citation data
- Enrich with CrossRef (if DOI available)
- Full pipeline via handler → papers stored in real SQLite
- Idempotent: second call doesn't duplicate
