# Plan 35-01: CLI Provider Registry + Batch Explain

## Goal

Add data-driven CLI provider registry for claude_cli/codex_cli, configurable fallback order, and batch explain to reduce LLM calls by ~10x during codegen.

## Tasks

### Task 1: CLI Provider Registry (DONE — commit 0031cbd)

**File created**: `shared/cli_providers.json` (~30 lines)
**File modified**: `shared/llm.py` (~130 LOC added)

- `_load_cli_configs()`: cached JSON loading
- `call_cli()`: generic CLI dispatcher (~40 LOC)

### Task 2: Batch Explain (DONE)

**File**: `services/codegen/explain.py` (~85 LOC added)

`explain_formulas_batch(formulas, batch_size=10)`:
- Chunks formulas into groups of `batch_size`
- Builds batch prompt: numbered formulas, requests JSON array
- Calls `fallback_chain()` once per chunk (1 call per 10 formulas)
- `_parse_batch_response()`: maps index to FormulaExplanation
- Fallback: if batch parse fails, returns empty dict (caller falls back to per-formula)

**File**: `services/codegen/main.py` (~15 LOC modified)

In `handle_process()`:
```python
batch_results = explain_formulas_batch(formulas)
for formula_row in formulas:
    fid = formula_row["id"]
    explanation = batch_results.get(fid) or explain_formula(...)
```
