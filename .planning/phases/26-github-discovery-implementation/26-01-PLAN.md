# Plan 26-01: GitHub Discovery Implementation

## Overview

Implement the GitHub Discovery feature designed in Phase 25. Refine the existing `github_search.py` head-start (297 LOC) into a production module, add SQLite schema extension, Pydantic models, and wire up POST /search-github + GET /github-repos endpoints on the orchestrator.

- Existing head-start: `services/orchestrator/github_search.py` (297 LOC)
- Shared library: `shared/db.py`, `shared/models.py`, `shared/llm.py`, `shared/server.py`
- Orchestrator: `services/orchestrator/main.py`

## Tasks

### Task 1: Schema Extension (`shared/db.py`)

**What**: Add `github_repos` and `github_analyses` tables + indexes to SCHEMA/INDEXES constants. Add migration from v1â†’v2 in `init_db()`.

**Changes**:
- Append `github_repos` CREATE TABLE to SCHEMA string (after `generated_code`)
- Append `github_analyses` CREATE TABLE to SCHEMA string
- Append 4 new indexes to INDEXES string
- Update `init_db()` to INSERT schema_version v2

**LOC**: ~30 lines added

### Task 2: Pydantic Models (`shared/models.py`)

**What**: Add `GitHubRepo`, `GitHubAnalysis`, `SearchGitHubRequest`, `SearchGitHubResponse` models.

**Changes**:
- `GitHubRepo`: paper_id, full_name, url, clone_url, description, stars, language, topics (JSON list), search_query
- `GitHubAnalysis`: repo_id, relevance_score, quality_score, formula_matches (JSON list), summary, recommendation, key_files (JSON list), dependencies (JSON list), model_used, analysis_time_ms, error
- `SearchGitHubRequest`: paper_id (required), max_repos=3, languages=["python","rust","cpp"], min_stars=5, query_override=None, force=False
- `SearchGitHubResponse`: paper_id, repos_found, repos_analyzed, results (list), errors (list)
- Use existing `_parse_json_list` for JSON field validators

**LOC**: ~70 lines added

### Task 3: Refine `github_search.py` Module

**What**: Major refactor of head-start into production module. This is the bulk of the work.

**Sub-tasks**:

3a. **Multi-language search**: `search_github()` takes `languages` list, builds per-language queries, merges+deduplicates by `full_name`, sorts by stars.

3b. **GitHub PAT auth**: Add `Authorization: Bearer $GITHUB_PAT` header + proper `X-GitHub-Api-Version` header. Read from `RP_GITHUB_PAT` env var, fallback to `GITHUB_PAT`.

3c. **Rate limiting**: Track `x-ratelimit-remaining` and `x-ratelimit-reset` from response headers. Sleep if remaining < 2.

3d. **Fix Gemini CLI invocation**: Use positional argument for prompt (not `-p`), use `--approval-mode yolo` (not `--yolo`). Already mostly correct in head-start but `-p` is used â€” fix to positional.

3e. **Multi-language file reading**: Extend `_read_repo_files()` with `EXTENSIONS = {"*.py", "*.rs", "*.cpp", "*.hpp", "*.c", "*.h"}` and `SKIP_DIRS` including `target`, `build`.

3f. **DB storage functions**: Add `_store_repo()` and `_store_analysis()` that INSERT into `github_repos` and `github_analyses` tables. Use `INSERT OR IGNORE` for repos (UNIQUE constraint on paper_id+full_name).

3g. **Paper context loading**: Add `_load_paper_context()` that queries papers + formulas from DB (reuse pattern from orchestrator's `_get_paper_detail`).

3h. **Rate limiting between Gemini analyses**: Sleep `60/RPM` seconds between each Gemini call. Read RPM from `RP_GITHUB_GEMINI_RPM` (default 5).

3i. **Refactor `search_and_analyze()`**: Accept `paper_id` + `db_path` instead of raw query. Load paper from DB, generate queries from title, search multi-language, clone + analyze top N, store results, cleanup.

3j. **Config env vars**: Read all `RP_GITHUB_*` vars per DESIGN.md spec.

**LOC**: ~350 lines (refactored from 297 head-start)

### Task 4: Orchestrator Endpoints (`services/orchestrator/main.py`)

**What**: Add POST /search-github and GET /github-repos route handlers.

**Changes**:
- `handle_search_github(data)`: Validate paper_id, call refactored `search_and_analyze()`, return `SearchGitHubResponse`
- `handle_github_repos()`: Parse query params (paper_id required, recommendation filter, limit), query DB, return list of repo+analysis pairs
- Import new models from `shared.models`

**LOC**: ~60 lines added

### Task 5: Extend `init_db()` Migration Path

**What**: Ensure `init_db()` handles both fresh databases (v1+v2 tables created) and existing v1 databases (only add v2 tables). The `CREATE TABLE IF NOT EXISTS` pattern handles this naturally.

**Changes**: Already covered by Task 1's schema additions â€” `CREATE TABLE IF NOT EXISTS` is idempotent.
