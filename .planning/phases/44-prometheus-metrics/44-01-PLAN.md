**4. Add GET /metrics route in `_register_builtins()`:**

Add alongside existing /health and /status registrations:

```python
from prometheus_client import generate_latest, CONTENT_TYPE_LATEST

@route("GET", "/metrics")
def handle_metrics(self_handler: BaseHandler) -> None:
    """Expose Prometheus metrics in text format."""
    output = generate_latest()
    self_handler.send_response(200)
    self_handler.send_header("Content-Type", CONTENT_TYPE_LATEST)
    self_handler.send_header("Content-Length", str(len(output)))
    self_handler.end_headers()
    self_handler.wfile.write(output)
```

IMPORTANT: This handler returns None (not a dict). The `_dispatch()` method already handles `if result is not None: send_json(result)` so returning None skips JSON serialization. The handler writes raw bytes directly with `text/plain; version=1.0.0; charset=utf-8` content type.

Bind it to handler class with the same pattern as handle_health and handle_status (check if not already defined by subclass).
