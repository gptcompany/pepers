## Tasks

### Task 1: Scoring Prompt Template

Design the exact prompt text for LLM paper analysis.

**Deliverable:** `services/analyzer/prompt.py` with:
- `PROMPT_VERSION = "v1"` constant
- `SCORING_SYSTEM_PROMPT` — system instruction for the LLM
- `format_scoring_prompt(title, abstract, authors, categories)` — builds the user prompt
- JSON output schema documented as docstring

**Prompt design principles:**
- System prompt sets role: "academic paper relevance scorer for Kelly criterion research"
- User prompt provides: title, abstract, authors, categories
- Output: strict JSON with 5 scores (0.0-1.0) + reasoning string
- Temperature 0.3 for reproducibility
- Max 500 tokens output

**5 criteria defined in prompt:**
1. `kelly_relevance`: Relevance to Kelly criterion, optimal bet sizing, portfolio allocation
2. `mathematical_rigor`: Formal proofs, derivations, theorems, mathematical notation
3. `novelty`: Original contribution beyond existing Kelly literature
4. `practical_applicability`: Real-world data, backtests, implementation guidance
5. `data_quality`: Dataset quality, methodology rigor, reproducibility

**Missing data handling:**
- If abstract is empty/None: score `kelly_relevance` and `novelty` as 0.0, skip LLM call, set overall to low score
- If title is empty: reject paper immediately (data quality too low)
- Prompt includes explicit instruction: "If the abstract is missing or very short (<50 chars), note this limitation in reasoning"

### Task 2: LLM Client Functions

Design the 3 client functions + fallback chain.

**Deliverable:** `services/analyzer/llm.py` with:

```
call_gemini_cli(prompt: str, system: str, model: str = "gemini-2.5-flash", timeout: int = 120) -> str
```
- subprocess.run with stdin=DEVNULL, GOOGLE_API_KEY env, -e none, --output-format json
- Raises RuntimeError on non-zero exit or API error
- Strips markdown fences from response field

```
call_gemini_sdk(prompt: str, system: str, model: str = "gemini-2.5-flash", timeout: int = 30) -> str
```
- google.genai Client with response_mime_type='application/json'
- Raises google.genai.errors.APIError on failure
- Returns response.text

```
call_ollama(prompt: str, system: str, model: str = "qwen3:8b", timeout: int = 120) -> str
```
- urllib.request POST to localhost:11434/api/generate
- format="json", stream=false, temperature=0.3
- Raises RuntimeError on non-200 or connection error
- Returns response["response"] field

```
fallback_chain(prompt: str, system: str) -> tuple[str, str]
```
- Tries: gemini_cli → gemini_sdk → ollama
- Returns (response_text, provider_name)
- Logs warning on each fallback
- Raises RuntimeError if all 3 fail

**Config needed:**
- `RP_ANALYZER_GEMINI_API_KEY` — Gemini API key (from SSOT)
- `RP_ANALYZER_GEMINI_MODEL` — default "gemini-2.5-flash"
- `RP_ANALYZER_OLLAMA_URL` — default "http://localhost:11434"
- `RP_ANALYZER_OLLAMA_MODEL` — default "qwen3:8b"

### Task 3: Schema Migration

Design the database schema change for prompt_version.

**Deliverable:** Migration in `shared/db.py` or `services/analyzer/main.py`:

```sql
ALTER TABLE papers ADD COLUMN prompt_version TEXT;
```

**Migration strategy:**
- Idempotent: check if column exists before ALTER
- Run at service startup (same pattern as init_db)
- Existing papers have NULL prompt_version (pre-analyzer)
- New index: `CREATE INDEX IF NOT EXISTS idx_papers_prompt_version ON papers(prompt_version)`

### Task 5: Handler Design

Design the AnalyzerHandler class and /process endpoint.

**Deliverable:** Design spec for `services/analyzer/main.py`:

```
class AnalyzerHandler(BaseHandler):
    @route("POST", "/process")
    handle_process(data: dict) -> dict
```

**Flow:**
1. Parse request (paper_id? max_papers? force?)
2. Query DB: papers WHERE stage='discovered' (or specific paper_id)
3. For each paper:
   a. Build prompt from title + abstract + authors + categories
   b. Call fallback_chain(prompt, system_prompt)
   c. Parse JSON response, validate 5 scores exist and are 0.0-1.0
   d. Compute overall = mean(5 scores)
   e. Update DB: stage='analyzed'|'rejected', score=overall, prompt_version='v1'
4. Return summary: papers_analyzed, papers_accepted, papers_rejected, avg_score, provider, errors

**Error handling per paper:**
- LLM failure (all 3 providers): log error, skip paper, add to errors list
- Invalid JSON from LLM: retry once with explicit JSON-only prompt suffix ("Respond ONLY with valid JSON, no markdown fences, no extra text"), then skip
- Invalid scores (out of range 0.0-1.0): log warning with actual values, clamp to 0.0-1.0, add "score_clamped" flag to paper's error field for traceability
