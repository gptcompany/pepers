# Phase 45, Plan 01: Docker Production Hardening

## Goal

Harden docker-compose.yml with log rotation, memory limits, graceful shutdown timeouts. Verify auto-start via Docker daemon systemd enablement.

## Prerequisites

- Phase 43 complete (ThreadingHTTPServer, SIGTERM handling)
- Phase 44 complete (Prometheus metrics endpoints)
- Docker daemon systemd-enabled: VERIFIED (enabled)

## Tasks

### Task 1: docker-compose.yml hardening

**Files to modify:** `docker-compose.yml`

**Changes:**

1. Add YAML extension fields at top of file (before `services:`):
   ```yaml
   x-logging: &default-logging
     driver: json-file
     options:
       max-size: "10m"
       max-file: "3"

   x-deploy-service: &default-deploy
     resources:
       limits:
         memory: 512m

   x-deploy-orchestrator: &orchestrator-deploy
     resources:
       limits:
         memory: 1g
   ```

2. Add to ALL 7 services:
   ```yaml
   logging: *default-logging
   ```

3. Add `deploy: *default-deploy` to 6 services (discovery, analyzer, extractor, validator, codegen, mcp)

4. Add `deploy: *orchestrator-deploy` to orchestrator

5. Add `stop_grace_period` to all services:
   - Regular services (discovery, analyzer, extractor, validator, codegen, mcp): `stop_grace_period: 10s`
   - Orchestrator: `stop_grace_period: 20s`

**Verification:**
```bash
docker compose -f docker-compose.yml config  # validates YAML syntax and anchor resolution
```

### Task 2: Validate config + test with dry run

**Steps:**

1. Run `docker compose config` to verify resolved YAML has correct values for all services
2. Run `docker compose build` to ensure images still build
3. Run `docker compose up -d` briefly and verify:
   - All containers start (health checks pass)
   - `docker inspect <container> --format '{{.HostConfig.LogConfig}}'` shows json-file with 10m/3 limits
   - `docker inspect <container> --format '{{.HostConfig.Memory}}'` shows correct memory limits (536870912 for 512m, 1073741824 for 1g)
4. Run `docker compose down` and verify completes within 30s
5. Write/update integration test that validates docker-compose.yml config programmatically

**Test file:** `tests/integration/test_docker_config.py`

Test assertions:
- All 7 services have `logging.driver == "json-file"` with max-size 10m, max-file 3
- All regular services have memory limit 512m (536870912 bytes)
- Orchestrator has memory limit 1g (1073741824 bytes)
- All services have stop_grace_period set
- `docker compose config` exits 0
