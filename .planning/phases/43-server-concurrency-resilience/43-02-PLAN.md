## Goal

Add stuck pipeline run cleanup at orchestrator startup, and write tests for all Phase 43 changes (threading, body limit, stuck-state).

## Files to Modify

### 1. `services/orchestrator/pipeline.py`

**New method: `PipelineRunner.cleanup_stuck_runs()`**

```python
def cleanup_stuck_runs(self, stale_minutes: int = 5) -> int:
    """Mark stuck 'running' pipeline runs as 'failed' on startup.

    A run is stuck if status='running' and started_at is older than
    stale_minutes ago (no completed_at).

    Args:
        stale_minutes: Minutes after which a running job is considered stuck.
            Default: 5. Configurable via RP_ORCHESTRATOR_STALE_MINUTES env var.

    Returns:
        Number of runs marked as failed.
    """
```

Implementation:
- Read `RP_ORCHESTRATOR_STALE_MINUTES` env var (default: stale_minutes param)
- SQL: `UPDATE pipeline_runs SET status = 'failed', errors = ?, completed_at = datetime('now') WHERE status = 'running' AND started_at < datetime('now', '-N minutes')`
- errors value: `'["Marked as failed: stuck in running state at service restart"]'`
- Log each cleaned run at WARNING level
- Return count of cleaned runs

### 2. `services/orchestrator/main.py`

**In `main()` function**, after `runner = PipelineRunner(config.db_path)`:

```python
# Clean up stuck pipeline runs from previous crashes
cleaned = runner.cleanup_stuck_runs()
if cleaned:
    logger.warning("Cleaned %d stuck pipeline run(s) from previous crash", cleaned)
```
