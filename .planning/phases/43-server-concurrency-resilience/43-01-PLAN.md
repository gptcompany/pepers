# Plan 43-01: ThreadingHTTPServer + Body Size Limit + Thread Safety

## Goal

Replace single-threaded HTTPServer with ThreadingHTTPServer, add 10MB body size limit checked via Content-Length header BEFORE reading body, and make handler class state thread-safe.

## Requirements Addressed

- **CONC-01**: ThreadingHTTPServer for concurrent requests
- **CONC-02**: 10MB body size limit with HTTP 413
- **CONC-03**: SQLite connections verified thread-safe (per-request)

## Files to Modify

### 1. `shared/server.py` (main change point)

**Change A: ThreadingHTTPServer**
- Import `ThreadingHTTPServer` from `http.server`
- In `BaseService.run()` line 318: replace `HTTPServer` with `ThreadingHTTPServer`
- Set `daemon_threads = True` on the server so threads don't block shutdown
- Update type hint on `BaseService.server` from `HTTPServer | None` to `ThreadingHTTPServer | None`

**Change B: Body size limit**
- Add constant `MAX_BODY_SIZE = 10 * 1024 * 1024` (10MB) at module level
- In `BaseHandler.read_json()`: BEFORE reading `self.rfile`, check `Content-Length` header against `MAX_BODY_SIZE`
- If `Content-Length > MAX_BODY_SIZE`: send HTTP 413 with error JSON `{"error": "Request body too large", "code": "PAYLOAD_TOO_LARGE", "details": {"max_bytes": 10485760, "received_bytes": N}}`
- If no `Content-Length` header on POST: already handled (returns 400 "empty body")

**Change C: Thread-safe handler state**
- Add `import threading` (already imported)
- Add class-level `_state_lock = threading.Lock()` on `BaseHandler`
- In `_dispatch()` line 119: wrap `self.__class__.last_request_time = time.time()` in `with self.__class__._state_lock:`
- Route cache `_routes` built lazily — already safe because dict assignment is atomic in CPython, but add lock for correctness

**Change D: Thread-safe route building**
- In `_dispatch()`, protect `_build_routes()` with the lock to prevent double-build race

### 2. `shared/db.py` (verification only, no code change needed)

- `get_connection()` creates NEW connection per call — already thread-safe
- `transaction()` context manager creates/closes per call — already thread-safe
- WAL mode allows concurrent reads — good
- Writes serialized at file level — acceptable for ~10 papers/day

**Action**: Add a comment at top of `get_connection()` docstring noting thread-safety guarantee: "Thread-safe: creates a new connection per call. Safe for use in threaded HTTP handlers."

## Implementation Steps

1. Add `MAX_BODY_SIZE` constant and import `ThreadingHTTPServer`
2. Add `_state_lock` to `BaseHandler`
3. Modify `read_json()` to check Content-Length before reading
4. Modify `_dispatch()` to use lock for `last_request_time` and route building
5. Modify `BaseService.__init__` and `run()` to use `ThreadingHTTPServer` with `daemon_threads = True`
6. Update `get_connection()` docstring in `shared/db.py`

## Verification

- `python -c "from shared.server import BaseService, BaseHandler; print('import ok')"` — no import errors
- Two concurrent `curl` requests to any service both succeed
- `curl` with Content-Length > 10MB returns 413
- All existing tests still pass

## Risk Mitigation

- Thread pool exhaustion: Not adding ThreadPoolMixIn now (YAGNI — ~10 papers/day, max 2-3 concurrent requests). If needed later, add in PERF-02.
- `daemon_threads = True` ensures clean shutdown even if threads hang.
