# Plan 15-01: CAS Microservice Implementation

## Overview

Implement the standalone CAS microservice at `/media/sam/1TB/cas-service/`. This is a separate repo with its own pyproject.toml, engines, and HTTP handler.

**Phase type**: Implementation
**Estimated LOC**: ~500

## Prerequisites

- [x] Phase 14 DESIGN.md complete
- [x] Maxima installed at /usr/bin/maxima

## Tasks

### Task 1: Initialize CAS repo

**Goal**: Create repo structure with uv, pyproject.toml, and directory layout.

**Steps**:
1. `mkdir -p /media/sam/1TB/cas-service/cas_service/engines`
2. `mkdir -p /media/sam/1TB/cas-service/tests`
3. Create `pyproject.toml` with sympy + antlr4-python3-runtime==4.11.1
4. Create `__init__.py` files
5. `cd /media/sam/1TB/cas-service && uv sync`

### Task 2: Implement engine abstraction + preprocessing

**Goal**: Create BaseEngine ABC, EngineResult dataclass, and LaTeX preprocessing.

**Files**:
- `cas_service/engines/__init__.py`
- `cas_service/engines/base.py` — BaseEngine ABC + EngineResult
- `cas_service/preprocessing.py` — 4-phase pipeline (strip envs, remove typographical, normalize, clean whitespace)

### Task 3: Implement SymPy engine

**Goal**: Implement SympyEngine with parse_latex + simplify.

**File**: `cas_service/engines/sympy_engine.py`

**Key behaviors**:
- parse_latex(latex, strict=True) with ANTLR backend
- Fallback to Lark backend if ANTLR fails
- Equation detection (split on `=`)
- 5s timeout via signal.alarm
- Return EngineResult

### Task 4: Implement Maxima engine

**Goal**: Implement MaximaEngine with subprocess execution.

**File**: `cas_service/engines/maxima_engine.py`

**Key behaviors**:
- LaTeX → Maxima syntax conversion
- subprocess.run with timeout
- Parse stdout for simplified/factored results
- Handle timeout, non-zero exit, empty output

### Task 5: Implement CAS HTTP handler + main

**Goal**: Create CASHandler with /validate, /health, /status, /engines endpoints.

**Files**:
- `cas_service/main.py` — CASHandler + main()
- `cas_service/__main__.py` — entry point

**Key behaviors**:
- POST /validate: parse request, preprocess LaTeX, run engines, return results
- GET /health, /status, /engines
- Engine registry initialized at startup
- JSON structured logging
- SIGTERM handling

### Task 6: systemd unit + smoke test

**Goal**: Create systemd unit file and verify the service starts and responds.

**Files**:
- `cas-service.service`
- Manual smoke test: start service, curl /health, curl /validate

## Acceptance Criteria

- [ ] `uv run python -m cas_service.main` starts on port 8769
- [ ] `GET /health` returns ok
- [ ] `POST /validate` with simple LaTeX returns SymPy + Maxima results
- [ ] Preprocessing strips environments and normalizes commands
- [ ] Engine errors return success=false with error message
- [ ] Timeout handling works for both engines
