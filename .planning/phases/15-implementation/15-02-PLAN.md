# Plan 15-02: Validator Service Implementation

## Overview

Implement the Validator service in `services/validator/` within the research-pipeline repo. Follows the existing service pattern (BaseHandler + @route).

**Phase type**: Implementation
**Estimated LOC**: ~350

## Prerequisites

- [x] Phase 14 DESIGN.md complete
- [x] Plan 15-01 (CAS microservice) — should be implemented first
- [x] shared/* library available

## Tasks

### Task 1: Create validator module structure

**Goal**: Create directory and files for the validator service.

**Files**:
- `services/validator/__init__.py`
- `services/validator/main.py`
- `services/validator/cas_client.py`
- `services/validator/consensus.py`

### Task 2: Implement consensus logic

**Goal**: Implement ConsensusOutcome enum, ConsensusResult, and apply_consensus().

**File**: `services/validator/consensus.py`

**Key behaviors**:
- 4 outcomes: VALID, INVALID, PARTIAL, UNPARSEABLE
- Decision matrix from DESIGN.md §2.5
- Works with any number of engines

### Task 3: Implement CAS client

**Goal**: HTTP client for CAS microservice using urllib.request.

**File**: `services/validator/cas_client.py`

**Key behaviors**:
- CASClient with validate() and health()
- EngineResult + CASResponse dataclasses
- CASServiceError exception
- Timeout handling (30s default)
- No external dependencies

### Task 4: Implement ValidatorHandler + DB operations

**Goal**: Main handler with /process endpoint and DB helper functions.

**File**: `services/validator/main.py`

**Key behaviors**:
- ValidatorHandler extends BaseHandler
- @route("POST", "/process") with batch processing
- _query_formulas(), _store_validations(), _update_formula_stage(), _mark_formula_failed()
- main() entry point with load_config + BaseService
- Summary report with counts (valid/invalid/partial/unparseable/failed)

### Task 5: Smoke test end-to-end

**Goal**: Verify the Validator service works with the CAS service.

**Steps**:
1. Start CAS service on :8769
2. Start Validator service on :8773
3. Ensure test formulas exist in DB
4. POST /process and verify response
5. Check validations table for results

## Acceptance Criteria

- [ ] `python -m services.validator.main` starts on port 8773
- [ ] `GET /health` returns ok
- [ ] `POST /process` with formula_id validates a formula
- [ ] Validation records written to DB (one per engine)
- [ ] Consensus logic correctly categorizes results
- [ ] CAS service errors are handled gracefully (formula marked failed)
- [ ] Empty batch returns success with formulas_processed=0
