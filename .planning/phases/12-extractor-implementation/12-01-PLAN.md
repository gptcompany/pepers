# Plan 12-01: Extractor Service Implementation

## Objective

Implement the Extractor service: PDF download from arXiv with retry/caching, RAGAnything HTTP client with polling, LaTeX regex extraction engine (5-pass with occupied-span tracking), HTTP handler, and DB storage for formulas.

## Implementation Steps

### Step 1: services/extractor/__init__.py — Empty init

Create empty `__init__.py` for package.

### Step 2: services/extractor/pdf.py — PDF Download

Per DESIGN.md Module 1:
- `EXPORT_BASE`, `USER_AGENT`, `DOWNLOAD_TIMEOUT` constants
- `create_session()` — `requests.Session` with `Retry(total=3, backoff_factor=2.0, status_forcelist=[429,500,502,503,504])`
- `get_pdf_url(paper)` — prefer stored `pdf_url`, rewrite to export domain, fallback to constructed URL
- `download_pdf(paper, dest_dir, session)` — download with cache check (skip if exists and >1000 bytes), stream download, content-type validation

Uses `requests` (already in pyproject.toml dependencies).

### Step 3: services/extractor/rag_client.py — RAGAnything Client

Per DESIGN.md Module 2:
- `DEFAULT_BASE_URL`, `DEFAULT_POLL_INTERVAL`, `DEFAULT_TIMEOUT` constants
- `check_service(base_url)` — check health + circuit breaker state (urllib.request)
- `submit_pdf(pdf_path, paper_id, base_url)` — POST to /process, handle 200 (cached) vs 202 (queued)
- `poll_job(job_id, base_url, timeout, interval)` — poll /jobs/{id} until completed/failed/timeout
- `read_markdown(output_dir)` — path mapping (container→host), find largest .md file
- `process_paper(pdf_path, paper_id, base_url)` — orchestration: check→submit→poll→read

Uses `urllib.request` (stdlib) — no extra deps.

### Step 4: services/extractor/latex.py — LaTeX Extraction Engine

Per DESIGN.md Module 3:
- `MATH_ENV_NAMES`, `MIN_FORMULA_LENGTH`, `CONTEXT_WINDOW` constants
- 5 compiled regex patterns (named envs → `\[..\]` → `$$..$$` → `\(..\)` → `$...$`)
- `extract_formulas(text)` — multi-pass with occupied-span set for overlap prevention
- `extract_context(text, start, end, window)` — surrounding text extraction
- `filter_formulas(formulas)` — remove short, non-math, and duplicate formulas
- `formulas_to_models(paper_id, text, raw_formulas)` — convert to `Formula` Pydantic models

Uses `re`, `hashlib` (stdlib) + `shared.models.Formula`.

### Step 5: services/extractor/main.py — Handler + main()

Per DESIGN.md Module 4:
- `ExtractorHandler(BaseHandler)` with class-level `max_papers_default`, `pdf_dir`, `rag_url`, `download_delay`
- `@route("POST", "/process")` — full flow:
  1. Parse request (paper_id, max_papers, force)
  2. Query papers with stage='analyzed'
  3. Check RAGAnything availability (abort batch on circuit breaker open → 503)
  4. For each paper: download_pdf → process_paper → extract_formulas → filter → store
  5. Rate limit between papers (download_delay)
  6. Return summary {papers_processed, formulas_extracted, errors}
- `_query_papers(db_path, paper_id, max_papers, force)` — query analyzed papers
- `_store_results(db_path, paper_id, formulas)` — insert formulas + update paper stage='extracted'
- `_mark_failed(db_path, paper_id, error)` — mark paper failed with error prefix
- `main()` — load_config, init_db, set handler attrs from env vars, BaseService.run()

## Risk Assessment

| Risk | Mitigation |
|------|-----------|
| RAGAnything not running | check_service() pre-batch, return 503 |
| arXiv rate limiting | 3s delay between downloads, Retry adapter |
| PDF not found (404) | Per-paper error, continue batch |
| RAGAnything job timeout | Per-paper error, continue batch |
| No formulas in paper | Valid outcome, paper still marked 'extracted' |
| Duplicate formulas | latex_hash deduplication in DB |
