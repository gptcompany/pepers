## Overview

Second microservice — LLM-based relevance scoring that filters discovered papers before expensive downstream processing (extraction, validation, codegen). Triple LLM fallback: Gemini CLI → Gemini SDK → Ollama qwen3:8b.

## Milestone Summary

**Key Decisions:**
- D-14: LLM Client — separate functions + fallback_chain() (KISS, each ~30 LOC)
- D-15: Scoring — simple mean of 5 equal-weight criteria (0.0-1.0 each)
- D-16: Prompt versioning — prompt_version column in papers table
- D-17: Threshold — 0.7 (restrittivo, ~60% papers filtered)
- REJECTED stage in PipelineStage — distinguish deliberate filtering from errors (FAILED)

**Issues Resolved:**
- Gemini CLI hangs without stdin=DEVNULL — fixed with subprocess flag
- google-genai SDK import — lazy import in call_gemini_sdk only (no hard dep)

**Issues Deferred:**
- Gemini API intermittent 503 errors — retry handled by fallback chain
- CAS microservice: only Maxima engine works (SageMath/MATLAB broken) — unrelated

**Technical Debt:**
- server.py run() method: 6 lines uncovered (process-level SIGTERM testing)
- call_gemini_sdk body untestable without real SDK — tested via mock
