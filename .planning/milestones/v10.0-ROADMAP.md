## Overview

Address 6 operational gaps discovered during the Phase 31 E2E smoke test (commit 1425c41) on real paper 2003.02743 (133 formulas, 107 codegen). The smoke test PASSED end-to-end but revealed resilience, reproducibility, and coverage gaps that prevent confident production deployment.

### Gap Origin

| # | Gap | Source | Severity |
|---|-----|--------|----------|
| 1 | Services crash under load, no auto-restart | smoke_test.py manual observation | HIGH |

| 6 | Extractor slow on CPU, no pre-caching docs | smoke_test.py extractor timeout 3600s | LOW |

## Phases

### Phase 32: Resilience — systemd Hardening

**Goal**: Ensure all 6 services survive crashes and maintain data consistency through restarts
**Depends on**: Phase 31 complete
**Research**: Unlikely (systemd patterns well-known)

Scope:
- systemd unit files: add `Restart=always`, `RestartSec=5`, `WatchdogSec=30`
- Add `/health` endpoint to all services (return process uptime, last request timestamp, DB connectivity)
- systemd watchdog integration: services notify sd_notify(WATCHDOG=1) on health check pass
- **Consistency check at startup**: each service verifies DB state on boot
  - Extractor: detect papers stuck in "extracted" with partial formula sets → re-extract or mark failed
  - Validator: detect formulas with partial validations (some engines done, others not) → re-validate
  - Codegen: detect formulas validated but missing generated_code rows → re-generate
- **UNIQUE constraint**: add `(paper_id, latex_hash)` to `formulas` table to prevent duplicate insertion after crash recovery (schema migration v3)
- Smoke test: kill -9 a service mid-batch, verify auto-restart and data consistency

Plans:
- [ ] 32-01: Schema migration v3 (UNIQUE constraint) + Docker restart + systemd units
- [ ] 32-02: Enhanced /health + startup consistency checks + resilience tests

### Phase 33: Reproducibility & Calibration

  - Update project structure to reflect actual directory tree
- Verification: run analyzer 3x on same paper, assert identical scores

Plans:
