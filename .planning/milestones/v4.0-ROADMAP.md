## Overview

Third microservice — extracts mathematical formulas from accepted papers. Downloads PDFs from arXiv, sends to RAGAnything for text extraction, parses LaTeX formulas via 5-pass regex engine with occupied-span tracking, stores formulas with context in SQLite.

## Milestone Summary

**Key Decisions:**
- PDF download: `requests` + `urllib3.Retry` (already in deps, full retry control)
- RAGAnything integration: sync polling not webhook (KISS, no concurrent state)
- LaTeX extraction: stdlib `re` multi-pass with occupied-span tracking (zero deps, 95% coverage)
- Formula filtering: hash dedup + min length + `\` check (removes false positives)
- Context window: 200 chars around each formula
- Schema: no changes needed (v1.0 `formulas` table already complete)
- Error handling: per-paper isolation (one failure doesn't block batch)
- Service architecture: same pattern as Analyzer (@route decorator)

**Issues Resolved:**
- Container→host path mapping for RAGAnything output directories
- Old-style arxiv IDs (hep-ph/0601001) → safe filenames with underscore
- LaTeX `$...$` false positives filtered by requiring `\` or `{` in content

**Issues Deferred:**
- Gemini API intermittent 429 errors (confidence gate rate-limited)
- CAS microservice: only Maxima engine works (SageMath/MATLAB broken)

**Technical Debt:**
- main.py startup code (main() function) uncovered in tests — process-level testing
- No SUMMARY.md for phases 12-13 (executed via pipeline, plan files serve as record)
