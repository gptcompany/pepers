## Milestone Summary

**Key Decisions:**
- SymPy `codegen()` for C99/Rust instead of manual string formatting — reliable, maintained
- Ollama-first fallback order for codegen (local, free, faster) vs Gemini-first for analyzer
- `parse_latex()` with ANTLR backend (more lenient than lark for real-world LaTeX)
- Per-language error isolation (one failure doesn't block others)
- FormulaExplanation as validation-only Pydantic model (JSON in formulas.description)
- LLM client extraction to shared/llm.py (reusable across services)

**Issues Resolved:**
- N8N W5.3 naive regex Rust codegen replaced with proper SymPy codegen("Rust")
- LLM code duplication between analyzer and codegen resolved via shared/llm.py
- Rust integer promotion #26967 documented as known SymPy issue

**Issues Deferred:**
- Rust integer promotion post-processing (SymPy upstream issue)
- MATLAB license temporarily unavailable (CAS has SymPy + Maxima)

**Technical Debt Incurred:**
- parse_latex brittle on non-standard LaTeX (per-formula error isolation mitigates)
- main.py coverage at 82% (main() entrypoint + some error paths uncovered)
