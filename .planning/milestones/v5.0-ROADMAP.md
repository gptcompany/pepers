# Milestone v5.0: Validator Service

**Status:** SHIPPED 2026-02-14
**Phases:** 14-16
**Total Plans:** 4

## Overview

Multi-CAS formula validation with all-or-nothing consensus. Includes a new standalone CAS microservice (SymPy + Maxima + MATLAB engines) and a Validator service that orchestrates validation, applies consensus logic, and stores results in SQLite. MATLAB engine temporarily unavailable (license expired, will return).

## Milestone Summary

**Key Decisions:**
- New CAS microservice in standalone repo (replaces broken N8N_dev version)
- SymPy + Maxima + MATLAB as CAS engines (SageMath dropped; MATLAB license temporarily unavailable)
- All-or-nothing consensus: both engines must agree for VALID
- stdlib urllib.request for CAS client (no new dependencies)
- UNPARSEABLE formulas stay as 'extracted' (not promoted to 'validated')

**Issues Resolved:**
- SageMath engine was broken in N8N_dev CAS service
- Original CAS service had no preprocessing pipeline
- No consensus logic existed in original pipeline

**Issues Deferred:**
- MATLAB engine license renewal (temporarily unavailable)
- RAGAnything isolation from N8N_dev (separate task)
- Old CAS service decommissioning (after MATLAB license restored)

**Technical Debt Incurred:**
- main.py coverage at 79% (entry point and generic exception handler uncovered)
- Maxima can disagree with SymPy on \cdot notation (LaTeX preprocessing gap)
