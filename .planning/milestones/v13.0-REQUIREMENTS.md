# Requirements: PePeRS v13.0

**Defined:** 2026-02-21
**Core Value:** Reliable, N8N-free academic paper processing pipeline

## v13.0 Requirements

Requirements for production hardening. All shipped.

### Concurrency

- [x] **CONC-01**: All 6 microservices handle concurrent requests via ThreadingHTTPServer
- [x] **CONC-02**: Request body size is limited to 10MB with 413 error on exceeding
- [x] **CONC-03**: SQLite connections are verified thread-safe (per-request, no caching)

### Monitoring

- [x] **MON-01**: Each service exposes /metrics endpoint with Prometheus format (request count, duration histogram, error count)
- [x] **MON-02**: Orchestrator exposes pipeline-specific metrics (papers processed, formulas validated, run duration)
- [x] **MON-03**: process-exporter identifies PePeRS services individually (not generic python-main.py)
- [x] **MON-04**: VictoriaMetrics scrapes all PePeRS /metrics endpoints (8770-8775)
- [x] **MON-05**: Grafana dashboard shows pipeline health (6 panels: service health, throughput, latency, errors, formula results, active runs)
- [x] **MON-06**: Grafana alert rules fire on service down (>3 min) and pipeline stalled (>24h no papers)

### Deployment

- [x] **DEP-01**: Docker logs rotate automatically (json-file, 10MB max, 3 files)
- [x] **DEP-02**: Docker containers have memory limits (512MB per service, 1GB orchestrator)
- [x] **DEP-03**: PePeRS Docker Compose auto-starts at boot (Docker daemon systemd-enabled)
- [x] **DEP-04**: Services shut down gracefully with stop_grace_period (10s services, 20s orchestrator)

### Resilience

- [x] **RES-01**: Orchestrator startup cleans stuck pipeline_runs (all running → failed with reason)
