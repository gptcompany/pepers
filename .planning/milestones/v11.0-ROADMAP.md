# Milestone v11.0: CLI Providers + Batch Explain + API + Async Orchestrator

**Status:** IN PROGRESS
**Phases:** 35-37
**Total Plans:** 0/3
**Depends on:** v10.0 (Phases 32-34) complete

## Overview

v10.0 shipped: 34 phases, ~12K LOC, 6 microservices, 682 tests. Pipeline works E2E.

**Problems v11.0 solves:**
1. Only 3 CLI LLM providers (Gemini CLI, OpenRouter, Ollama) — user wants Claude CLI and Codex CLI
2. Codegen explanation calls LLM 1x per formula (133 formulas = 133 calls)
3. No GET /generated-code — code only accessible nested via /papers
4. POST /run is synchronous — blocks for hours, client timeouts

**Approach:** Data-driven CLI provider registry (JSON config), generic `call_cli()`, batch explain, async /run with polling.

## Milestone Success Criteria

- [ ] `RP_LLM_FALLBACK_ORDER=claude_cli,gemini_cli` works with fallback

## Breaking Changes

- POST /run returns HTTP 202 (async) instead of 200 (sync)
- Clients must poll GET /runs?id=xxx for results
