## Milestone Summary

**Key Decisions:**

- APScheduler over host cron: pure Python, travels with Docker stack, full env var access
- network_mode: host: external services (RAG:8767, CAS:8769, Ollama:11434) on host, simplest networking
- Shared SQLite volume: sequential pipeline = no concurrent writers, WAL mode safe
- Cron disabled by default: on-demand via POST /run, user opted for manual/batch triggers

**Issues Resolved:**

- DB stage name mismatch: "discovered" vs "discovery" — fixed with DB_STAGE_INDEX mapping
- Port 8775 conflict during testing — killed lingering process
- BaseService /status override: renamed method to handle_status
- Return type annotation: handle_run returns dict | None

**Issues Deferred:**

- Monitoring integration: process-exporter config, Prometheus alert rules, Grafana dashboard
- Production Docker deployment: `docker compose up` on Workstation

**Technical Debt Incurred:**

- None significant — clean implementation following established patterns
