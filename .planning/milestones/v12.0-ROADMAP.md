# Milestone v12.0: Distribution & Branding

**Status:** IN PROGRESS
**Phases:** 38-42
**Total Plans:** 2/5
**Depends on:** v11.0 (Phases 35-37) complete

## Overview

v11.0 shipped: 37 phases, ~13K LOC, 6 microservices, 728+ tests. Pipeline works E2E with async /run, batch explain, CLI providers.

**Problems v12.0 solves:**
1. No project identity — still named "research-pipeline", no branding, no logo
2. RAGAnything POST /query calls internal LLM (30-90s) — MCP clients could synthesize faster
3. No MCP Server — Claude Desktop/Cursor can't access pipeline tools natively
4. No one-click install — requires manual systemd/Docker setup
5. Only arXiv discovery — misses papers from other sources (biomedicine, CS, engineering)

**Approach:** Brand first (avoid refactoring later), then RAGAnything context_only (prerequisite for fast MCP), then MCP SSE wrapper, then Docker packaging, finally OpenAlex multi-source.

- [x] 39-01: context_only mode in RAG wrapper + orchestrator (6 new tests, 705 total)

### Phase 40: MCP Server SSE

**Goal**: MCP Server (SSE transport) wrapping orchestrator HTTP API, exposable to Claude Desktop/Cursor/etc.
**Depends on**: Phase 39 (context_only for fast search)
**Research**: Likely (MCP SDK Python, SSE transport patterns)

Scope:
- `services/mcp/server.py` — Python `mcp` SDK, SSE transport, ~250 LOC
- 7 tools: search_papers, list_papers, get_paper, get_formulas, run_pipeline, search_github, get_generated_code
- systemd unit `rp-mcp.service` on port 8776
- `uv add mcp` dependency
- **Arcade flavor output** (configurabile via `RP_MCP_FLAVOR=arcade|plain`):
  - Tool descriptions con tono gaming (es: "Scout mode: RAG-powered paper search")
  - Output prefix con emoji arcade per ogni tool response:
    - Search: `[PePeRS] Target locked. {n} papers found. FIRE!`
    - CAS validation: `[PePeRS] S-RANK ACHIEVED! The math holds up.` / `[PePeRS] SHORYUKEN! CAS validation failed.`
    - Codegen: `[PePeRS] Blanka-Shock! Code is now executable.`
    - Pipeline run: `[PePeRS] MISSION COMPLETE. 1 CREDIT(S) well spent.`
  - Flavor dict separato (~30 LOC) per facile personalizzazione
  - Default: arcade. `RP_MCP_FLAVOR=plain` per output sobrio

Plans:
- [x] 40-01: MCP Server SSE + arcade flavor + deploy (server.py 427 LOC, 40 new tests, 745 total)
